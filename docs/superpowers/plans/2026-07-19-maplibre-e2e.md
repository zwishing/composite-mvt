# MapLibre End to End Verification Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a manually runnable, local-only MapLibre GL JS and Playwright verification path that proves `composite-mvt` serves one gzip-compressed composite MVT containing queryable `roads` and `buildings` features.

**Architecture:** Keep all server behavior in the example layer. A small Rust example loads deterministic fixture bytes and same-origin static assets at startup, caches them in memory, and serves the page, local MapLibre distribution, and z0 tile with `tiny_http`. The browser page uses an empty MapLibre style and a direct vector `tiles` template. Playwright drives the real browser and asserts the tile response, MapLibre status, queried source-layer counts, browser errors, and the separate tile failure state.

**Tech Stack:** Rust 2024 with MSRV declaration 1.87, `composite-mvt`, `fast-mvt = 0.6.0` writer and reader APIs, `tiny_http = 0.12.0`, native HTML/CSS/JavaScript, `maplibre-gl = 5.24.0`, `@playwright/test = 1.61.1`, and Chromium installed by Playwright.

- [ ] **Step 2: Implement the minimum writer-based fixtures**

Use the verified `fast_mvt 0.6.0` builder types already available through dev dependencies. The implementation must use `MvtTileBuilder`, `MvtGeometry`, `MvtLineString`, `MvtPolygon`, and `MvtCoord`, not manually assembled protobuf bytes:

```rust
use std::error::Error;
use std::fs;
use std::path::Path;

use fast_mvt::{
    DEFAULT_EXTENT, MvtCoord, MvtGeometry, MvtLineString, MvtPolygon, MvtResult,
    MvtTileBuilder,
};

fn roads_tile() -> MvtResult<Vec<u8>> {
    let geometry = MvtGeometry::LineString(MvtLineString::new(vec![
        MvtCoord { x: 512, y: 2048 },
        MvtCoord { x: 3584, y: 2048 },
    ]));
    let mut layer = MvtTileBuilder::new().layer_with_capacity("roads", 1)?;
    layer.extent(DEFAULT_EXTENT);
    let mut feature = layer.feature(&geometry)?;
    feature.id(Some(101));
    feature.tag_string("kind", "primary")?;
    Ok(feature.end().end().encode())
}

fn buildings_tile() -> MvtResult<Vec<u8>> {
    let exterior = MvtLineString::new(vec![
        MvtCoord { x: 1024, y: 1024 },
        MvtCoord { x: 3072, y: 1024 },
        MvtCoord { x: 3072, y: 3072 },
        MvtCoord { x: 1024, y: 3072 },
    ]);
    let geometry = MvtGeometry::Polygon(MvtPolygon::new(exterior, Vec::new()));
    let mut layer = MvtTileBuilder::new().layer_with_capacity("buildings", 1)?;
    layer.extent(DEFAULT_EXTENT);
    let mut feature = layer.feature(&geometry)?;
    feature.id(Some(202));
    feature.tag_string("kind", "office")?;
    Ok(feature.end().end().encode())
}

fn main() -> Result<(), Box<dyn Error>> {
    let directory = Path::new(env!("CARGO_MANIFEST_DIR"))
        .join("examples")
        .join("maplibre")
        .join("fixtures");
    fs::create_dir_all(&directory)?;
    fs::write(directory.join("roads.pbf"), roads_tile()?)?;
    fs::write(directory.join("buildings.pbf"), buildings_tile()?)?;
    Ok(())
}
```

```javascript
"use strict";

const status = document.querySelector('#status');
if (!(status instanceof HTMLElement)) {
  throw new TypeError('missing #status element');
}

const setStatus = (state, roads, buildings, message) => {
  status.dataset.state = state;
  status.dataset.roads = String(roads);
  status.dataset.buildings = String(buildings);
  status.textContent = message;
};

const map = new maplibregl.Map({
  container: 'map',
  style: {
    version: 8,
    sources: {
      composite: {
        type: 'vector',
        tiles: ['/tiles/{z}/{x}/{y}.pbf'],
        minzoom: 0,
        maxzoom: 0,
      },
    },
    layers: [
      { id: 'buildings-layer', type: 'fill', source: 'composite', 'source-layer': 'buildings', paint: { 'fill-color': '#4f46e5', 'fill-opacity': 0.55 } },
      { id: 'roads-layer', type: 'line', source: 'composite', 'source-layer': 'roads', paint: { 'line-color': '#111827', 'line-width': 3 } },
    ],
  },
  center: [0, 0],
  zoom: 0,
  minZoom: 0,
  maxZoom: 0,
  attributionControl: false,
});

map.on('idle', () => {
  if (status.dataset.state === 'ready' || status.dataset.state === 'error') {
    return;
  }

  const roads = map.querySourceFeatures('composite', { sourceLayer: 'roads' }).length;
  const buildings = map.querySourceFeatures('composite', { sourceLayer: 'buildings' }).length;
  if (roads > 0 && buildings > 0) {
    setStatus('ready', roads, buildings, `ready: roads=${roads}, buildings=${buildings}`);
  }
});

map.on('error', (event) => {
  if (status.dataset.state === 'ready') {
    return;
  }

  setStatus('error', 0, 0, event.error?.message ?? 'MapLibre error');
});
```

The error listener must not overwrite a later successful status in normal operation, but the tile-abort E2E must observe `data-state="error"`.
